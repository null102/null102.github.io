二.

我刚写完上一篇文章没多久，工具确实不是最重要的东西，最重要的是脑壳里面装着的那团有精神的糊糊，它能手动或者自动的不断迭代，产生许多不可思议的东西。我们现在所享用的所有工具，都是从我们祖先那些脑壳里的糊糊里面产生的。我对工具有很多感受，不涉及任何技术细节我就能就关于工具写出一本书来，但是最根本的，知道什么是好东西就行了，然后去探究里面的思想和技术上的细节，我对许多工具背后的思想略知一二，但是我对它技术上的细节很多都不甚了解。了解了之后就会发现很多东西并不是完美的，但是我觉得我们生活的真实世界本质就是残缺，有一个不残缺的世界在宇宙或精神中的某个地方，它是完美的，但是它是更加矛盾的。妥协了矛盾我们就只能生活在残缺的世界里，所有人的使命都是将残缺的世界补全，感受到自己使命的有那么些人，真正或多或少做到的或许能被列在一本几十或者几百页的书上，真正或多做到的大概一张纸上就能列完了。但是补完了又有什么用，世界永远会被平衡在那个残缺程度的状态，我不知道这个世界在往哪里堆，是变大还是变小，不过我暂时假设它残缺的比例是永恒的。

我能给出的建议是，热爱自己做的东西，并且有充分理由的热爱自己的工具。不一定所有的开发者都要用 Vim 和 Emacs，但是我觉得这些都是至少要学一下的，它们里面有些机制很垃圾，但是这两个是最有哲理并且最为有机的编辑器，Visual Studio 还有 JetBrains 很强大，但是他们显得太人造了。所有音乐制作人至少都要用 Reaper 一段时间，并且能够使用各种 Tracker ，这样的 DAW 能够帮助人理解编曲的本质和编曲的有机性。所有使用计算机的人至少要有把所有操作系统都用一遍的意向，至少有时间和精力要去配置自己的lfs。Windows 和 Mac OS 都不完美，甚至大部分时间都显得很垃圾，但是 Linux 和 BSD 的底层也都是错误百出甚至有时又臭又长的，这些系统有生态，但是比较一下，它们其实还不如 Oberon 要先进，不过说实话我到现在用过体验最好的系统是索尼这几世代基于 BSD 开发的那些用于 PlayStation 的系统。写作者都要试着用 LaTex，它代表了计算机系统里最高层次的排版。网上冲浪者要把所有浏览器内核和排版系统都用一遍，就像冲浪者要找到一块好的冲浪板一样。探索工具是有趣的，不说 Vim 和 Emacs 这种民用级别高层次的工具，即使是 DSA 键帽的 Cherry MX2 或者 Fl Studio 这样稍微有意思一点的工具，我相信已经比你身边那些烂逼的脑糊糊要好玩一万倍了。
